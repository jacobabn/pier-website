Built for the messy stuff

Modern AI for sites without a modern stack

No Git. No CI/CD. No build pipeline. Just FTP and a 12-year-old PHP file. Pier brings AI workflows to the websites the rest of the dev world forgot about.

Talk, don’t click

"Update the homepage hero to say X." "Find every page using the old logo." Pier finds the right file, makes the right change, and shows you the diff before applying.

Versioned everything

Every accepted edit becomes a checkpoint. Restore any file to any state in one click. Auto-snapshot before risky changes (.htaccess, wp-config). Nothing is ever truly broken.

Read your code, propose surgical edits

Pier never blindly rewrites files. It does a find/replace with surrounding context, validates the match is unique, and shows the diff. You stay in control.

Built-in audit agents

Type @seo, @security, @performance, @broken_links — Pier scans the live site and reports issues. Each one comes with a one-click "fix it for me" proposal.

Drag & drop, anywhere

Drop files into the chat — Pier uploads them and asks the AI what to do with them. Drop into the Files tab to upload to a specific folder. Works the way you expect.

Knows your site

On connect, Pier scans the server, detects the CMS (WordPress / Drupal / Magento / Joomla / custom PHP), reads the .htaccess, and remembers everything across chats.

The old way vs. the Pier way

You used to FTP into the server, hunt for the file, copy it locally, edit it in your favorite editor, save, upload, refresh, hope nothing broke. Pier is one chat message.

FAQ

Questions, answered

Do I need an Anthropic API key?

No. Pier works with either your own Anthropic API key (pay per token, ~$0.01-0.05/turn) or your existing Claude Code subscription (zero per-turn cost). It auto-detects what you have. We’ll guide you through setup the first time.

Can it actually edit my live website?

Yes — that’s the whole point. Pier connects via FTP/SFTP to any host (DirectAdmin, cPanel, Plesk, custom). Every edit is proposed as a diff first — you click Accept, Pier uploads, and the file’s previous state is saved as a version you can restore at any time.

Is my FTP password safe?

Yes. Credentials are encrypted (AES-256-GCM) on disk with HMAC integrity checks. Nothing leaves your Mac except the FTP traffic itself. We never see your passwords. The encryption key is stored in your local .env with owner-only file permissions and benefits from FileVault disk encryption.

What if the AI breaks my site?

Two safety nets: (1) every edit is a diff you approve before applying, (2) every accepted edit creates a restore point. For high-risk files (.htaccess, wp-config.php, composer.json) Pier auto-creates a folder snapshot before applying. Worst case: one click and you’re back to the previous state.

Does it work with WordPress?

Excellent with WordPress. Pier detects WP automatically, knows the theme/plugin/uploads layout, and reads wp-config.php safely (secrets are auto-redacted in chat). For content stored in the database (posts, pages), Pier tells you so directly — you’ll need wp-admin or phpMyAdmin for those. Pier handles every theme file, plugin file, and config that lives on disk.

What about agencies / teams?

Each Pier seat handles unlimited connections. Add a license per developer; everyone gets their own chat history per client site. Team-shared knowledge base is on the roadmap.

Can I cancel anytime?

Yes — cancel from the Mac App Store subscription manager. You keep access until the end of your current billing period. Lifetime purchases are non-cancellable but include all updates for the current major version.

Why is it called Pier?

A pier is where boats dock to load and unload cargo. Your AI assistant docks with the FTP server to deliver and retrieve files. Short, memorable, and the metaphor actually works.
